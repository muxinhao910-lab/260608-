# 造物任务入口

## 当前任务

验收并继续聚焦 `/admin/builder` 的真实可用性。

请先根据用户手动验收结果判断 Builder 是否满足最小闭环：

- 能看到组件库
- 能添加标题
- 能添加段落
- 能添加按钮
- 能添加 Banner
- 能添加信息卡片
- 能添加板块入口卡片
- 能添加分割线
- 能点击选中模块
- 能编辑文字
- 能编辑链接
- 能保存
- 刷新后内容还在
- 能重置

如果发现问题，请把问题拆成小范围 Builder 修复任务交给 Codex。

## 输入来源

- `docs/agents/current-goal.md`
- `docs/agents/decision-log.md`
- 本角色 inbox 文件

## 输出位置

请将结果写入 `docs/agents/outbox/zaowu-report.md`。

## 执行边界

不要越权执行其他角色职责。

## 结束规则

完成任务后，必须在对应 outbox report 末尾写明“下一步交接”。

如果你能判断下一步角色，请给出下一步角色和可复制指令。

如果你不能判断下一步角色，请明确写：

“请交回司南判断下一步。”

## 角色边界

造物负责 `/admin/builder`、可视化组件编辑器、组件库、添加组件、选中模块、属性面板、编辑文字 / 链接 / 图片 / 布局、保存、刷新保留、重置、后台工具能力，以及未来低代码 / 模块化编辑能力。

造物不负责股票数据源，不负责 Git 安全，不负责整体产品路线。

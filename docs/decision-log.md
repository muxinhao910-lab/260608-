# Decision Log

## Decision 001 - GitHub is the source of project truth

Chat history is not the source of truth.

The repository documentation is the source of truth.

Important context, decisions, role boundaries, and current project status should be written into docs.

## Decision 002 - Developer Mode means Builder

Developer Mode should not be treated as a generic admin dashboard.

Developer Mode means entering `/admin/builder`, where the user can visually add and edit website components.

## Decision 003 - The user is the idea owner, not a professional PM

The user can describe ideas informally.

The product manager role should translate informal ideas into structured requirements, Codex tasks, and acceptance criteria.

## Decision 004 - Functional correctness before visual restoration

The website should eventually have strong visual impact and scroll-driven effects.

But current priority is functional correctness, routing, Builder usability, and safe development workflow.
